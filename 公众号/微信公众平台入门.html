                # 微信公众账号简介

                微信用户--->微信 APP -> 微信公众账号

                微信 App 是平台, 微信用户通过微信 APP 来订阅微信公众账号, 获取相关资讯和服务

                微信公共账号作为自媒体, 可以将网站应用, 移动端应用 通过微信平台, 提供给微信用户使用, 同时推送自身一些信息给用户

                # 公众账号分类: 服务号, 订阅号, 企业号

                http://kf.qq.com/faq/120911VrYVrA130805byM32u.html 1、订阅号：主要偏于为用户传达资讯（类似报纸杂志），认证前后都是每天只可以群发一条消息； 2、服务号：主要偏于服务交互（类似银行，114，提供服务查询），认证前后都是每个月可群发 4 条消息； 3、企业号：主要用于公司内部通讯使用，需要先有成员的通讯信息验证才可以关注成功企业号；

                订阅号: 通讯录 -> 订阅号 , 微信应用里面, 所有的订阅号, 都是放在订阅号这个栏目里面, 要打开订阅号这个栏目后才能看到里面有哪些订阅号, 点击订阅号进去后, 下面有功能菜单, 还可以发送消息给订阅号, 除了下面的菜单按钮外, 右上角是公众号介绍, 其他部分界面类似一般的聊天界面

                服务号: 服务号就直接处于微信的界面的首页, 进去后显示的是相关的推送的图文消息, 右上角是公众号介绍, 下面也是菜单, 进去菜单后是自定义的服务..

                企业号: 企业号在通讯录页面, 在一般联系人的最上面就是企业号

                # 公众账号认证

                http://kf.qq.com/faq/120911VrYVrA130805byM32u.html

                价格 300 元/次 每年, 认证和不认证的区别是可用的接口不同

                注册公众账号, 用未注册过的邮箱来注册, 主体类型可以是政府媒体企业其他组织, 个人 但除个人外, 都需要其他信息如营业执照等. 个人注册的话, 只需要身份证信息即可....

                还需要, 该公众号的运营者的手机号码

                申请微信公众号 mp.weixin.qq.com

                比如:

                申请邮箱: huizhibangong@163.com huizhibangong123 13917661850

                公众账号: huizhibangong@163.com

                运营者微信认证: 13917661850 运营者手机: 13917661850

                # 编辑模式和开发者模式

                公众账号使用模式分为编辑模式和开发者模式

                公众账号后台 , 功能 区域的内容就是编辑模式可以使用的功能, 编辑模式就是开发者模式下的部分可视化操作

                测试了下发现, 微信的编辑模式下功能很不错了, 自动回复, 自定义菜单都可以在线编辑

                但是自定义菜单如果需要跳转网页的话必须要经过认证才行

                服务号的话可以开通微信支付, 有一些高级接口

                订阅号, 想实现在线订单支付肯定不可能, 但是不知道能不能实现认证后跳转到网页, 在线下单.....

                另外, 订阅号 , 自定义菜单后需要 10 小时后才能生效

                群发消息, 需要微信管理者账号扫二维码, 而且不是马上就发送出去(正在排队等候系统发送，可能需要 10-20 分钟)

                那么, 订阅号, 更多的应该是用在介绍商品上面, 就好比是网站 , 主要可以介绍服务, 经营的商品, 然后就是自动回复和群发消息

                最后一个就看是否能实现在线订单了 ...... 这个应该是可以的, 因为 我们的第三方服务器 理论上是可以读取到微信客户信息的,

                这样就可以实现第三方服务器中保存订单

                那么先看看简单的功能和开发环境的搭建:

                # 2. 验证服务器

                https://mp.weixin.qq.com/wiki?t=resource/res_main&id=mp1421135319&token=&lang=zh_CN

                验证消息的确来自微信服务器 开发者提交信息后，微信服务器将发送 GET 请求到填写的服务器地址 URL 上，GET 请求携带参数如下表所示：

                signature 微信加密签名，signature 结合了开发者填写的 token 参数和请求中的 timestamp 参数、nonce 参数。 timestamp 时间戳 nonce 随机数 echostr 随机字符串

                在验证的时候, 微信 SDK 里面的例子不能验证通过, 必须要有输出 echo $_GET['echostr']; 才能验证通过 另外 就是需要后退重新开启, 再添加配置 要退出重新来一次, 才验证成功 经测试, 国外 IP 域名也可以验证通过

                        if ($tmpStr == $signature) {
            echo $_GET['echostr'];
            return true;
        } else {
            return false;
        }

                # 关注/取消关注事件的自动回复

                https://mp.weixin.qq.com/wiki?t=resource/res_main&id=mp1421140454&token=&lang=zh_CN

                在微信用户和公众号产生交互的过程中， 用户的某些操作会使得微信服务器通过事件推送的形式通知到开发者在开发者中心处设置的服务器地址， 从而开发者可以获取到该信息。

                关注/取消关注事件 用户在关注与取消关注公众号时，微信会把这个事件推送到开发者填写的 URL。 方便开发者给用户下发欢迎消息或者做帐号的解绑。 微信服务器在五秒内收不到响应会断掉连接，并且重新发起请求，总共重试三次。

                这个 URL 是我们自定义的, 在上一步中验证的时候用的也是这个 URL , 所以这个 URL 中需要实现验证, 然后判断是否是第一次 验证的时候, 服务器端是会发送 echostr 的 , 如果判断 echostr 为空, 那么就说明当前收到的内容是微信服务器发送过来的用户事件
