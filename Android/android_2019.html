      # 理解 compileSdkVersion , targetSdkVersion, Gradle 和 SDK 版本等

      https://chinagdg.org/2016/01/picking-your-compilesdkversion-minsdkversion-targetsdkversion/

      很常见的情况是下载下来的一些 andriod 项目, 它默认的 build.gradle 的 compileSdkVersion 在本地开发环境并没有下载对应版本的 sdk

      且项目的 project construre 因为构建错误, 无法用界面方式修改 compileSdkVersion ,

      其实 compileSdkVersion 并没有那么重要, 一般只要本地安装的 sdk 版本比它项目原本的版本高, 可以直接修改配置文件,

      改成使用我们更高的 compileSdkVersion 版本来编译, build-tools 版本号需要进入到 sdk 路径后, 然后 cd build-tools,

      ls 查看一下你都下载了哪些 buildtools 的版本。如果没有你想要的，可以通过 Android SDK Manager 进行下载.
