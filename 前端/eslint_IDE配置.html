            # 参考资源

            慕课网 vue 核心技术中的 eslint 章节 react 文档: 配置 ide eslint 基础 http://www.cnblogs.com/yzg1/p/6282791.html editorconfig http://www.cnblogs.com/xiyangbaixue/p/4201490.html prettier https://prettier.io/

            # 工具介绍:

            eslint -- 定制代码规则 editorconfig -- 帮助开发人员定义和维护一致的编码风格在不同的编辑器和 IDE prettier -- 代码格式化

            editorconfig 一般如果这个项目没有配置 eslint 的话, 也就是没有规范化代码的规则的时候, 才会使用 editorconfig, 在 idea 下默认的快捷是 opt + cmd + l

            prettier 是专注于格式化代码, 而且还可以根据 eslint 规则来格式化代码

            但 eslint 也可以调用插件, 然后 当代码风格不符合规则, 这几种工具都能通过配置来实现自动修复的功能

            # vscode

            https://egoist.moe/2017/12/11/write-better-code-with-eslint-and-prettier/ https://nice.lovejade.cn/zh/article/beautify-vue-by-eslint-and-prettier.html#%E6%B7%BB%E5%8A%A0%E7%BC%96%E8%BE%91%E5%99%A8%E9%85%8D%E7%BD%AE https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode

            # 使用 preitter 结合 eslint 规则来格式化代码

            // Format a file on save. A formatter must be available, the file must not be saved after delay, and the editor must not be shutting down.

            "editor.formatOnSave": true,

            表示使用配置的 formatter 来格式化代码, 于是在 vscode 中, 需要安装 prettier 插件, 然后再配置, prettier 使用 eslint 规则

            "prettier.eslintIntegration": true,

            这样就可以在保存的时候格式化代码了

            # vscode 使用 eslint 来自动修复代码

            但是... eslint 也有格式化的功能, 比如在 vs code 中安装了 eslint 插件后 , 可以配置

            "eslint.autoFixOnSave": true,

            这两种方法都可以格式化, 但我一般用 prettier 更多 , 因为 prettier 可以格式化很多其他格式的代码, 比如 json 文件, 还有其他格式的文件

            而且 这两个都开启为 true 发现一般都是用的是 prettier 生效

            # prettier

            filewatcher 结合 prettier 使用

            1. 全局安装 sudo yarn global add prettier

            2. 在 intellij 中安装 file watcher 插件 , 并在 intellij 中设置 file watcher

            那么这个是 prettier 规则的格式化代码, 而不是 eslint 规则, 所以 intellij 中不推荐使用,

            而在 vscode 中推荐使用 prettier , 因为它使用 eslint 规则很方便而且支持多种文件格式

            # reformate code

            但 intellij 中还有个 reformate code , 它默认用的是 editorconfig 配置文件 快捷键是 opt + cmd + l

            如果有 filewatcher + prettier 同时存在的话, 那么 editorconfig 可能和 prettier 规则冲突

            包括 intellij 的自动补全( shift + cmd + 回车 )

            在自动补全后, 会自动保存, 而在保存前又要去调用 prettier , 所以即使有冲突但最终使用的是 prettier

            一般情况下如果没有 eslint 的情况下, 那么才使用这个 opt + cmd + l 来 reformate code

            # eslint 结合 fix eslint problem

            在 intellij 中安装了 javascript support 默认安装 , 那么它提供了 fix eslint problem 这个功能

            但需要注意的是, 要开启该功能才能使用, 而且该项目本身是有安装 eslint 依赖且配置了 eslint 配置文件 ,

            eslint 配置文件可以是单独的 eslintrc 这样的文件, 也可以是 package.json 中的 eslintConfig 字段

            然后每个项目都需要手动去开启,在设置中搜索 eslint -> enable 即可

            可以直接给这个功能设置快捷键, 比如 opt + l

            在 react 项目中使用 eslint , create-react-app 创建的项目 eject 后可见依赖包中是有安装 eslint 还有相关的规则配置的

            只要 yarn run start 的时候, eslint 是可以正常工作的, 在开启 intellij 中的 eslint 后 fix eslint problem 这个功能就可以正常工作

            推荐使用该方式, 虽然不是保存时候自动修复问题, 但是使用快捷键修复也已经是很方便了, 而且使用的是 eslint 规则来修复, 符合我们的需求

            包括补全符号啊, 这些都可以修复

            # prettier 结合 git 在代码提交前调用 prettier 格式化代码

            另外 react 官方推荐的是使用 prettier 来实现自动修复,

            但解决方案是和 git 关联的, 在文件提交前那么就调用 prettier

            https://facebook.github.io/create-react-app/docs/setting-up-your-editor

            # intellij 中使用 prettier

            使用 eslint 的时候 比如 html 等等都不能很好的格式化, 而 prettier 可以格式化更多文件格式

            比如格式化 html 还可以格式话 markdown 文件

            全局安装 sudo yarn global add prettier

            /usr/local/lib/node_modules/prettier

            然后在 ide 的设置中设置 prettier 所在路径

            修改快捷键, 默认是三个键的组合快捷很不方便, 改成 opt + p

            而我们前面有提到使用 eslint 自动修复的快捷是 opt + l

            这两个快捷就比较一致好记了,

            当需要使用 prettier 就是 opt + p

            当需要使用 eslint 那么久是 opt + l

            使用编辑器的规则就是 opt + cmd + l

            # 总结

            目前看来最好的方式还是在 ide 中保存的时候就调用 eslint 规则来进行修复

            只要项目中配置了 eslint 依赖和配置文件, 然后打开 intellij 的 eslint fix 功能 , 结合快捷键还是很方便的

            而在 vscode 中, 则使用 prettier 集成 eslint 规则也很方便
