          # vue press

          # 资源

          尤神自己介绍 vuepress https://www.youtube.com/watch?v=lIv1ItUzktc&t=714s

          这篇对侧边栏等介绍的比较清楚 https://segmentfault.com/a/1190000017055963 https://www.yzblog.xyz/vuepressblog.html

          yarn global add vuepress 或 npm install -g vuepress

          安装完成即可用vuepress , 它有三个参数, vuepress dev -- start development server vuepress build -- build dir as static site vuepress eject -- 拷贝默认的主题到 .vuepress/theme

          默认主题就是为文档主题 支持markdown语法

          比如:

          在项目目录准备一个readme.md文件(因为默认首页是 readme.md),

          vuepress dev 运行后查看

          支持热加载, 修改 readme.md 页面会马上加载变化

          需要注意的是多的使用 md 语法, 一些以前很少用到的样式都可以使用, 比如引用样式平时就很少用

          但 vuepress 能把这些样式显示的很美观, 比如

          // 引用
> markdown

// 提示框
::: tip
this is tips!
:::

// link链接
[go to other page](./hello.md)

          # link 链接

          [go to other page](./hello.md)

          实际上生成的是单页面应用, 并没有实际的跳转整个页面, 它只是在本页面内跳转,

          这样速度更快, 而且 vuepress 可以有很好的 SEO

          # 定制样式

          项目目录下的.vuepress 文件夹存放样式主题

          配置文件config.js

          module.exports = {
  title: 'Hendry 笔记汇总',
  themeConfig: {
    nav: [{ text: 'Another page', link: '/hello.html' }],
    sidebar: ['/', '/hello']
  }
}

          在 md 文档顶部定义本页面自动生成sidebar 本页面的 header 自动生成 sidebar 不要 sidebar 的话就设置 sidebar:false

          # 上传到 github

          创建 github 博客页面, 成功后可以通过https://ruiqingzheng.github.io/ 来访问创建的对应的仓库

          https://blog.csdn.net/Maple_ROSI/article/details/79484691

          然后可以把仓库下载到本地测试下, 在仓库里放上一些页面上传后浏览主页看是否成功更新

          但每次 vuepress 生成的文件是在.vuepress/dist 目录, 所以我们只能每次在 vuepress build 后把更新 push 到服务器

          https://zhuanlan.zhihu.com/p/36390666

          在脚本里先进入到 vuepress 的输出目录, 然后调用 git 命令

          这里使用 git 上传每次都重新初始化了仓库, 然后强制覆盖远程仓库, 所以 github 上看起来只有一个 commit 提交, 而我们一般也不需要关心它的历史版本

          #!/usr/bin/env sh
​
# abort on errors
set -e
​
# build
vuepress build
​
# navigate into the build output directory
cd .vuepress/dist
​
# if you are deploying to a custom domain
# echo 'www.example.com' > CNAME
​
git init
git add -A
git commit -m 'deploy'
​
# if you are deploying to https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master
​
# if you are deploying to https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages
​git push -f git@github.com:ruiqingzheng/ruiqingzheng.github.io.git master
cd -

          设置使用 ssh key 验证上传, 在 github 页面的 settting 里面设置 id_rsa.pub 公钥 然后把以前的私钥放到 ~/.ssh/id_rsa 并设置权限 chmod 700 id_rsa

          这样就以后上传代码就不需要输入密码了, 而且这里是对 bitbucket 也是一样的 ssh key

          如果 key 不同的话, 那么需要管理多个 ssh key , 比如针对 github 使用一对 , 使用 bitbucket 使用另外一对

          具体的操作方法也不难, 参考这篇文章管理多个 ssh key

          https://www.jianshu.com/p/194f787998c1
