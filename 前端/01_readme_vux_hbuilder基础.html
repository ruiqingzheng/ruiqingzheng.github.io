                      # 真机调试

                      https://segmentfault.com/a/1190000011182088

                      android 真机调试用的都是adb 而hbuilder的adb 在它的安装目录下C:\WORKSPACE\soft\HBuilder.9.0.2.windows\HBuilder\tools\adbs

                      调用命令

                      adb kill-server
adb start-server
adb devices

                      手机必须要打开开发者模式和USB调试还有手机驱动 , 这3点缺一不可

                      1. 开启开发者模式: 关于手机 -> 版本号 点击7次开启开发者模式
                      2. 开启USB调试: 辅助功能-> 开发者选项 -> USB调试
                      3. 电脑端安装手机驱动 安装好驱动后设备管理器可见 android adb 设备 魅族安装驱动, 需要先设置为光盘连接
                      4. usb调试授权, 弹窗点击允许即可
                      5. 奇怪的是魅蓝note2 必须要在电脑端安装豌豆荚才能正常使用hbuilder真机调试 暂时先就这样

                      # 后退问题

                      https://blog.csdn.net/qq_25252769/article/details/76913083

                      打包后的应用依然是在webview中运行, hbuilder 打包后的应用依然是在浏览器核心中运行

                      但这个浏览器内核被称为是 html5plus中的webview是对原生webview的封装

                      它和普通的浏览器略有不同, js可以调用api来访问html5plus的webview 也就是hbuilder打包后的webview对象

                      这个对象实际上就很类似浏览器的window对象 , 但它封装了更多功能, 可以实现一些手机端的功能, 比如访问手机按钮事件, 手势操作, 截屏操作

                      Webview模块管理应用窗口界面，实现多窗口的逻辑控制管理操作。通过plus.webview可获取应用界面管理对象。

                      https://zhaomenghuan.github.io/2016/05/19/mui%E5%88%9D%E7%BA%A7%E5%85%A5%E9%97%A8%E6%95%99%E7%A8%8B%EF%BC%88%E4%BA%8C%EF%BC%89%E2%80%94html5+%20webview%20%E5%BA%95%E9%83%A8%E6%A0%8F%E7%94%A8%E6%B3%95%E8%AF%A6%E8%A7%A3/#webview%E5%9F%BA%E6%9C%AC%E7%9F%A5%E8%AF%86

                      http://www.html5plus.org/doc/zh_cn/webview.html

                      那么我们使用hbuilder打包 , 就可以使用该webview对象, 且通过该对象来实现对手机后退按钮的支持, 在网上可以找到一些例子中还用到mui , 但我们实际用的是vue + vux

                      因此, 只需要webview对象及其, 其他的mui的代码是不需要的

                      http://ask.dcloud.net.cn/docs/#http://ask.dcloud.net.cn/article/91
