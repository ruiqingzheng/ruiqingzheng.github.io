      # 第二章 eslint

      # 2-4 eslint

      # eslint 基础

      eslint 规则

      eslint 官方提供了 3 种预安装包: eslint-config-google, eslint-config-airbnb, eslint-config-standard

      如果是 standard 配置的话 参考https://www.npmjs.com/package/eslint-config-standard

      安装 eslint npm install eslint --save-dev

      手动模式使用 standard-config, 安装依赖包

      npm install --save-dev \
    eslint-config-standard \
    eslint-plugin-standard \
    eslint-plugin-promise \
    eslint-plugin-import \
    eslint-plugin-node

      添加 eslint 对 vue 的支持

      npm install -D eslint-plugin-html

      // .eslintrc

      {
  "extends": "standard",
  "plugins": [ "html" ]
}

      # 配置 package.json 使用命令 eslint 和修复

      修改 package.json 定义命令执行 lint 检查, 和 lint --fix 自动修复错误

      // package.json

          "lint": "eslint --ext .js --ext .jsx --ext .vue client/",
    "lint-fix": "eslint --fix --ext .js --ext .jsx --ext .vue client/",

      但这只是用 npm run lint 或 npm run lint-fix 就能调用 eslint ,

      但如果不手动调用的话, webpack 在打包的时候是不会使用 eslint 检查代码的

      # 配置 webpack 的 eslint

      npm install eslint-loader babel-eslint

      修改 eslint 配置文件 , 项目代码都是会被 babel 处理 , webpack 使用 eslint-loader

      如果不设置 parser 为 babel-eslint 那么会有一些问题

      // .eslintrc

      {
  "extends": "standard",
  "plugins": ["html"],
  "parser": "babel-eslint"
}

      // webpack.config.js

      定义在 vue-loader 之前使用 eslint 预处理, enforce: 'pre' , post 指的就是之后再处理

          rules: [
      {
        test: /\.(vue|js|jsx)$/,
        loader: 'eslint-loader',
        exclude: /node_modules/,
        enforce: 'pre'
      },
      {
        test: /\.vue$/,
        loader: 'vue-loader',
        options: createVueLoaderOptions(isDev)
      },

      # 使用 editorconfig 插件

      不同的编辑器有不同风格, 可以通过使用 editorconfig 插件让编辑器代码风格一致

      http://www.cnblogs.com/xiyangbaixue/p/4201490.html

      editorconfig 会自动根据规则来修复代码风格, 比如加上空行等

      # 使用 husky 来规范 git 提交前 lint 代码

      https://www.jianshu.com/p/f0d31f92bfab

      需要先初始化仓库, 安装 husky , 它是 git 的钩子工具, 当运行 git commit 的时候会先调用我们定义的命令

      // package.json

          "lint": "eslint --ext .js --ext .jsx --ext .vue client/",
    "lint-fix": "eslint --fix --ext .js --ext .jsx --ext .vue client/",
    "precommit": "npm run lint-fix",

      这样当运行 git commit 的时候, 真正提交之前会 , husky 会帮我们先调用 npm run precommit

      也就是 npm run lint-fix 来 lint-fix 代码, 这样就能强制规范代码后提交代码

      # eslint 和 prettier 结合 vscode

      https://egoist.moe/2017/12/11/write-better-code-with-eslint-and-prettier/

      而 vscode 插件 prettier 默认安装了 prettier-eslint 插件, 而且 vscode 的 prettier 插件会在

      editor.formatOnSave 开启的话, 自动调用 prettier,

      而且 prettier 默认是读取自己的配置文件, 比如.prettierrc, .prettierrc 等

      如果不存在 prettier 配置文件, 那么它会读取 editorconfig 的配置文件 .editorconfig和 vscode 的 prettier 配置

      如果需要使用 prettier-eslint 格式化, 而不是 prettier 的规则去格式化代码,

      则需要在 vscode 中设置 "prettier.autofix on save": true,

      但需要注意的是 prettier-eslint 并不是说按 eslint 去格式化代码

      prettier-eslint 只是 Formats your JavaScript using prettier followed by eslint --fix

      也就是说启用 vscode 的 eslint 插件也有个选项是 autofix on save的话, 它最终还是要按 prettier 规则去格式化代码

      如果最终需要 prettier 格式化代码

      那么开启 prettier.eslintIntegration 和 editor.formatOnSave 则可以实现最终代码一定是 prettier 风格格式

      如果最终需要是 eslint 格式的话 , 那么完全可以不安装 prettier

      vscode 的 eslint 插件也有个选项是 autofix on save 开启这个选项即可

      # prettier

      https://egoist.moe/2017/12/11/write-better-code-with-eslint-and-prettier/ https://nice.lovejade.cn/zh/article/beautify-vue-by-eslint-and-prettier.html#%E6%B7%BB%E5%8A%A0%E7%BC%96%E8%BE%91%E5%99%A8%E9%85%8D%E7%BD%AE https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode

      eslint-plugin-prettier does not install Prettier or ESLint for you. You must install these yourself.

      npm install --save-dev eslint-plugin-prettier npm install --save-dev --save-exact prettier

      Then, in your .eslintrc.json:

      {
  "plugins": ["prettier"],
  "rules": {
    "prettier/prettier": "error"
  }
}
