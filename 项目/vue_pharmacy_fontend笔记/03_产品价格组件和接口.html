                    # price-packages组件的数据更新

                    然后发现price-packages数据没有动态更新, 最后发现必须父子组件中同时设置watch

                    父组件中当路由发生变化, 那么就改变绑定的属性变量, 比如这里的this.name_key = product_name; 相当于通知子组件更新

                    子组件中检测该属性pname判断是否发生变化, 从而加载数据

                    // 父组件中:

                        watch: {
      // 如果路由有变化，会再次执行该方法
      '$route' (to, from) {
        // 对路由变化作出响应...

        let product_name = this.$route.params.name;
        this.name_key = product_name;
        this.getProduct(product_name);
      }
    },

                    // 子组件中:

                        watch: {
      // 如果路由有变化，会再次执行该方法
      'pname' () {
        // 对路由变化作出响应...
        console.log("pname changed:" + this.pname);
        if (this.pname.length > 0) {
          this.getPackages(this.pname);
        }
      }

                    # 引入组件的问题

                    import {PricePackages} from '@/components/price-packages/price-packages';

                    这样用大括号引入的话, 那么源文件中必须是 export PricePackages ,

                    而如果源文件是 export default , 那么就不能用大括号 , 简单说 , 大括号表示明确引入 有名称定义的方法或对象

                    # 子组件数据的更新依赖父组件的watch

                    父组件传递属性给子组件, 该属性是父组件的路由参数, 子组件直接watch该属性是无法正常获取到变更的

                    子组件数据的更新依赖父组件的watch 中, 对该属性变量的修改

                    虽然, 最终效果是可以正常更新数据, 但这样是不是不改合理, 因为父组件还需要对属性进行watch 来通知子组件
