                  # element Menu 的使用

                  其中需要特别注意 v-for 遍历对象

                  https://cn.vuejs.org/v2/guide/list.html

  </div>

                  # 需指定key

                  当 Vue.js 用 v-for 正在更新已渲染过的元素列表时，它默认用“就地复用”策略。如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序， 而是简单复用此处每个元素，并且确保它在特定索引下显示已被渲染过的每个元素。这个类似 Vue 1.x 的 track-by="$index" 。

                  这个默认的模式是高效的，但是只适用于不依赖子组件状态或临时 DOM 状态 (例如：表单输入值) 的列表渲染输出。

                  为了给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，你需要为每项提供一个唯一 key 属性。理想的 key 值是每项都有的且唯一的 id。这个特殊的属性相当于 Vue 1.x 的 track-by ，但它的工作方式类似于一个属性，所以你需要用 v-bind 来绑定动态值 (在这里使用简写)：

                  # mapActions或mapMutations时如何传参

                  https://github.com/vuejs/vuex/issues/579

                  调用的时候, 直接传参就可以了, 不用担心第一个参数会被覆盖,

                  比如这里getProductAction: function (context , name) , 它的第一个参数context 是不会被覆盖的

                      methods: {
      ...mapActions({
        getProduct: "getProductAction"
      }),
    }

    created() {
      let product_name = this.$route.params.name;
      console.log(product_name);
      if (product_name) {
        this.getProduct(product_name);
      }
    },

                  # 路由切换后, 重新加载数据

                  按官方的来

                  https://router.vuejs.org/zh-cn/essentials/dynamic-matching.html

                  使用watch 或者 beforeRouteUpdate

                  在watch中, 直接写方法

                      watch: {
      // 如果路由有变化，会再次执行该方法
//      '$route' : getProduct
      '$route' (to, from) {
        // 对路由变化作出响应...
        let product_name = this.$route.params.name;
        this.getProduct(product_name);
      }
    },

                  # 图片

                  在vue中使用图片 静态加载 http://blog.csdn.net/letasian/article/details/76219889

                  但我们这里并不是要静态的加载图片 , 而是把图片从远程显示出来, 所以, 图片也是放在后台, 或者放在某个远程地址

                  然后把图片的地址用img标签加载出来即可, 图片是不存在跨域的

                  1. 我们把原来AAA站点的图片都拷贝到 后台站点的 api_images 目录下, 这样我们在vue中就可以加载了

                  2. vue中显示图片

                    // :src 后面字符串中写表达式即可

                    <img alt="" :src="image_root + product.image_prefix + '/' + product.image0">

                  3. 图片显示封装为统一方法

                    然后我们把图片的地址, 提取为方法:

                          image_url(image_key) {
        if (image_key.length > 0) {
          return this.image_root + this.product.image_prefix + '/' + image_key;
        }
      }

                  # 注意数据是否加载

                        <div v-if="product.image3.length > 0">
        <h3>image3</h3>
        {{product.image3}}
      </div>

                  这里图片是异步远程获取的, 这样判断的话, 当使用到这些数据的时候, 就有可能报错, 因为product很可能还没有加载成功, 那么它就是未定义的, 报错 product undefined

                  然后我们把store中的判断产品是否加载的字段映射过来

                      computed: {
      ...mapGetters({
        product: "productInfo",
        productLoaded: "isProductLoaded",
      }),

                  这样就可以在表达式中先判断数据是否加载, 再使用数据了

                        <div v-if="productLoaded && product.image0.length > 0">
        <h3>image0</h3>
        <img alt="" :src="image_url(product.image0)">
      </div>
