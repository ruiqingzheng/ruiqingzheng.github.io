                # 产品详情页面

                思路:

                实现无跳转的产品详情页 , 当在分类页面点击产品后, 那么在当前页面上显示一个新的层 (z-index设置覆盖即可), 这个新的层用来显示产品数据

                这样做的好处是, 没有页面的跳转, 而且当用户点击返回直接回到前面的分类页所在的位置

                要实现这样的效果, 用的是vue-router的子路由来实现

                比如我们的项目中

                1. 在 分类页面 添加显示区域 <router-view></router-view>

                2. 修改router , 给分类页面加上子路由

                  // 原本的做法 , 会跳转页面

                      {
    path: "/product/:product_key",
    component: Product,
    props: true
    },

                  // 修改产品页面, 作为分类页面的子路由

                  {
    path: "/category",
    component: Category,
    children : [
        {
        path: ":product_key",
        component: Product,
        props: true
        }
    ]
    },

                3. 分类页面设置点击事件, 跳转路由

                  
<li @click="selectProduct(product,$event)"
...

    // 事件处理 跳转子路由
    selectProduct(product) {
    console.log(product);
    this.$router.push({
        path: `/category/${product.product_id}`
    })
    }

                4. 子路由组件的样式, 设置全屏覆盖

                  • z-index, 在上面已经制定了子路由组件是Product组件, 那么这个组件最终显示的时候, 需要设置z-index 大于分页页面的z-index
                  • position: fixed , 左上角位置, 而且需要设置position

                基本上上面就已经可以实现需要的效果了 , 另外, 因为这种结构比较常用, 在分页跳到产品页面详情 , 我们还需要加载动画效果

                于是, 我们再新建一个product-detail组件, 让product 组件去加载product-detail组件

                product组件接收路由参数id, 然后负责加载数据, 加载数据后传递给product-detail组件进行显示, 于是全屏覆盖的样式设置放到product-detail

                而product组件的样式也需要设置, 使用vue animation内置的动画设置页面跳转动画

                // product 组件 , 负责动画加载页面, 和接收路由参数加载数据, 然后调用实际的显示组件ProductDetail

                <template>
  <transition name="slide">
    <ProductDetail :product="product_detail">    </ProductDetail>
  </transition>
</template>

// 定义的name 是slide , 于是根据这个动画的name, 用css 样式来设置动画
<style lang="stylus" rel="stylesheet/stylus" scoped>
  .slide-enter-active, .slide-leave-active
    transition: all 0.3s
  .slide-enter, .slide-leave-to
    transform: translate3d(100%, 0, 0)
</style>

                // ProductDetail组件

                
    <template>
    <div class="product-detail-container">
        <span style="color: red;">hello world, {{product.product_id}}</span>
    </div>
    </template>

// 设置全屏覆盖样式

    <style lang="stylus" rel="stylesheet/stylus" scoped>
    @import "~common/stylus/variable"
    .product-detail-container
        z-index :100
        position : fixed
        top: 0
        right:0
        bottom:0
        left:0
        background: $color-background
    </style>

                # 注意z-index 是个相对值

                比如整个详情页面最外层元素, 我们设置了z-index 100 , 而详情页面实际上只是分类页面的一个router-view 页面元素 这个页面中的router-view中的元素z-index比其他平级的也就是分类的ui内容z-index要高, 所以我们的detail页面覆盖了分类页面

                而在我们的详情页面中, 其他子元素之间又是一个相对的z-index, 也就是说在详情页面的z-index重新开始, 详情页面中的所有元素的z-index都是0

                比如我们希望在详情页面中的返回按钮, 希望它在所有元素之上, 只需要设置它的z-index : 1 , 即可

                而后来, 我们要在整个页面上顶部显示产品型号, 为了方便居中显示, 我们让这个标题宽度为100% , 因为也在顶部, 100%于是覆盖了后退按钮

                这时点击后退按钮就能触发事件 , 于是又要设置该标题元素的z-index , 这个z-index 应该要比后退按钮小即可 , 比如后退设置为2, 这里设置为1即可.

                其他的就是产品组件的设计和显示了 , 另外后退, 直接使用 this.$router.back() 就能退回上一页, 而且是无刷新, 位置也保存的... 所以这个方式的确是用户体验非常好

                # 产品图片展示

                使用vux的previewer组件 https://doc.vux.li/zh-CN/components/previewer.html

                参考他们demo , 其中图片的显示可以只显示一张图片

                我们最终效果需要让它像背景图片一样, 点击后能展示一些图片

                // template

                
    <div class="detail-header" v-if="previewer_list && previewer_list.length>0">
      <img  :src="previewer_list[0].src" @click="show(0)">
     
      <div v-transfer-dom>
        <previewer :list="previewer_list" ref="previewer" @on-index-change="logIndexChange"></previewer>
      </div>
    </div>

                // script

                    methods: {
        show(index){
            this.$refs.previewer.show(index)
        },
    },
    watch: {
      product(){
        this.previewer_list =  this.product.images.map(function (image) {
          return {src: image}
        })
      }
    },
    directives: {
      TransferDom
    },
    components: {
      Previewer
    },

                需要注意的是header 图片展示, 这个图片的高度是从原始图片得来, 因为原始图片是250px, 所以才设置的 .product-header: 250px

                # scroll

                产品展示图片, 应该也被包含在scroll里面, 就是说整个页面都应该放在scroll中

                如果展示图片在头部不动的话, 其实头部图片就占用了太多空间, 不利于产品信息的展示

                而更好的一种解决方案是让头部图片保留一部分 , 进行推拉的控制顶部区域的大小, 但这个实现比较复杂可以留在升级版本中进行

                所以暂时我们把整个页面容器作为scroll

                # position 和 scroll

                position fixed 就不显示滚动条 position absolute 就显示滚动条

                另外就是如果把scroll作为最外层的元素的话, 那么页面切换动画又会导致问题

                scroll 放在最外层的里面就不会有切换动画的问题了

                最后发现不用betterscroll , 直接使用浏览器自身自动产生的scroll已经够用

                https://css-tricks.com/scroll-fix-content/ https://blog.csdn.net/u014520745/article/details/52881300
