            # 成功案例页面

            数据整理来说也是类似二维数组, 按案例城市列表划分, 每个城市下案例是个列表

            在显示上, 我们让城市做一个类似tab的菜单导航放在顶部

            内容区域显示每个城市下的案例, 用时间轴改装成的列表来显示

            每个案例只有标题和简短说明, 另外配上几个缩略图片, 点击图片可以看大图

            暂时不做每个案例的详情页面, 因为数据不全拿不到数据, 而且时间有限, 这个可以以后改进

            # CaseRegionsHeader.vue

            用的vux swiper来做这个头部地区显示 , 每个swiper-item 里面放3个地区菜单

            于是把获取到的数据需要分成3个一组

            数据的获取和分组都放在store里面, 虽然页面分成了头部和body两个部分,而且两个部分的某些数据是关联的, 比如点击头部菜单, 需要获取对应的 body内容

            那么数据传递原本是需要事件来处理, 或者是父子组件属性传递数据, 但用了store数据的传递就不是问题了, 菜单中点击触发后, 获取到的数据, body列表可以直接去store中获取即可

            其他的就是布局和样式了

            # body部分

            body对应的是 CaseList.vue 组件

            设计是将time-line的设计, 有css-demo

            数据的获取 ,CaseList 中的数据, 实际都是在CaseRegionsHeader mounted 时候来抓数据 , 因为用的是store来获取数据

            所以CaseList中直接使用这些数据即可, 在初始化的时候, 我们需要获取地区列表 (提供给头部菜单使用), 然后获取第一个地区的case数据(提供给列表使用)

            // store vcase.js

                // 初始化获取地区列表, 然后获取第一个地区的case数据
    async initCases_action({commit}) {
      try {
        let regions_response = await getCaseRegionsApi();
        commit("set_caseRegions", regions_response.data.data);
        let default_region = regions_response.data.data[0];
        let cases_response = await getRegionCasesApi({id: default_region._id});
        commit("set_regionCases", cases_response.data.data);
      } catch(error) {
        console.log("initCases_action , error: ", error);
      }
    }

            当头部菜单被点击, 需要切换菜单样式, 于是对原有的数据, 需要加上属性selected

            而且, 每当点击菜单的时候都需要去获取该region下的cases数据这些直接看代码.

            另外, time-line样式中, 图片点击后展示图片, 用的是vux previewer , 它需要的数据中必须有src属性来显示previewer

            于是store中的getter对数据region_cases 加上src属性.

            # Tips

            # vue 动态样式class

            可以使用v-bind直接把元素的class设置为对象 , 对象的key 就是样式class , 值是true, false控制是否添加该class

            <div class="static"
     v-bind:class="{ active: isActive, 'text-danger': hasError }">
</div>

data: {
  isActive: true,
  hasError: false
}

结果渲染为：

<div class="static active"></div>
